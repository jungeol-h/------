# 데이터 모델링 및 시연 시나리오 (Mock 기반)

본 단계에서는 DB 없이 프론트엔드 코드 내 `mockData.js` 단일 파일로 모든 데이터를 통제한다.

## 1. Mock 데이터 모델링 구조
최소한의 M:N(다대다) 관계를 보여주기 위해 아래와 같은 하드코딩 객체를 선언해 둔다.

```typescript
// 1. 가상의 학생들
const mockStudents = [
  { id: 's1', name: '김안동', grade: '중2', selfIndex: 85, isWarning: true },
  { id: 's2', name: '이수진', grade: '중3', selfIndex: 92, isWarning: false }
];

// 2. 학생이 남긴 가상의 오늘 기록
const mockRecords = [
  { studentId: 's1', type: '학습', content: '수학 1단원', time: '120분', focus: 3 },
  { studentId: 's1', type: '마인드', mood: '힘듦', stress: '높음' }
];

// 3. 가상의 교육자 계정
const mockEducators = [
  { id: 'e1', name: '최강사', role: '교과강사', assignedStudents: ['s1', 's2'] },
  { id: 'e2', name: '박매니저', role: '학습매니저', assignedStudents: ['s1'] }
];
```

## 2. 권장하는 발표(시연) 플로우
이 프로토타입을 통해 사업 심사 시 다음과 같은 흐름으로 앱을 작동시키며 발표하는 것을 권장한다.

1. **학생 시점 시연:** 
   * "먼저 학생 시점입니다." 라며 [학생 퀵-로그인] 버튼 클릭
   * 학습 탭에서 타이머를 누르고, 마인드 탭에서 '오늘 기분: 힘듦'을 클릭하여 저장하는 척 액션.
2. **매니저(코치) 시점 시연:** 
   * 로그아웃 후 [학습매니저 퀵-로그인] 버튼 클릭
   * 대시보드 진입 시, 방금 학생이 '힘듦'을 체크한 내용이 즉각적으로 붉은색 알림(위험 감지)으로 떠 있는 것을 보여줌. (상태 공유 시연)
   * 클릭하여 "응원 코멘트"를 남김.
3. **학부모/관리자 시점 시연:**
   * 로그아웃 후 [관리자 퀵-로그인] 버튼 클릭
   * 누적된 거대한 통계 차트 화면과 함께 "이처럼 기록된 데이터는 사업 운영 성과 리포트로 즉시 PDF 변환이 가능합니다." 멘트 후 인쇄 버튼(`window.print()`) 시연.
