# 1차 프로토타입 개발 계획

## 1. 개발 핵심 목표 (가설 검증 및 시연)
이번 1차 프로토타입의 절대적인 목표는 상급자/심사위원에게 **"본 플랫폼이 어떤 동선과 가치를 가지고 동작하는지 시각적으로 완벽히 설득하는 것"**이다.

* **복잡도 배제 (Logic-Less):** AI 에이전트를 활용하면 다량의 화면(UI)과 컴포넌트를 빠르게 찍어낼 수 있다. 이 강점을 활용해 5개의 직군(학생, 학부모, 강사, 매니저, 컨설턴트)의 대시보드와 화면 이동 동선을 **전부 다 양적으로 구현**한다. 반면, 복잡한 통계 계산식, DB 스키마 정규화, 토큰 기반 인증 등 눈에 보이지 않는 백엔드 로직은 전면 배제하거나 더미(Mock)로 대체한다.
* **시각적 완성도 집중:** 시스템이 실제로 방대한 데이터를 처리하는 것처럼 보이게 만드는 데 집중한다. 차트 라이브러리(Chart.js, Recharts 등)를 적극 활용해 화려하고 꽉 찬 대시보드를 시연하는 것이 핵심이다.

## 2. 기존 레포(CLAUDE 진단 사이트) 연동 전략
기존에 개발되어 있던 진단 사이트 레포지토리는 핵심 로직이 이미 존재하므로, 이것과 신규 시스템을 통합해야 하는 과제가 있다. 하지만 1차 프로토타입의 제한된 시간 안에서 물리적인 DB 마이그레이션이나 레포 통합을 시도하는 것은 극도로 위험하다.

### **프로토타입 적용 전략: 프론트엔드 뷰(View) 껍데기만 이식**
1. **완전 분리 및 UI 흉내내기:** 신규 프로토타입 프로젝트에 기존 진단 로직을 가져오지 않는다. 대신 신규 앱의 <진로 탐구 탭>에 `[진단 시작하기]` 버튼을 둔다.
2. **시연용 하드코딩 플로우:** 이 버튼을 누르면 기존 진단 사이트의 프론트엔드 UI만 흉내낸 더미 설문 화면(1~2문항)이 나오고, 완료 시 곧바로 **가짜 데이터로 꽉 채워진 '진단 결과 리포트' 화면**으로 넘어간다.
3. **향후(2단계) 실제 통합 방향:** 실제 본 개발에 착수할 때는 신규 플랫폼과 기존 진단 서버를 API 형태(Microservice)로 통신하게 하거나, 혹은 아예 신규 레포를 Monorepo(모노레포)로 설정하여 기존 컴포넌트를 안전하게 이주(Migration)시킬 예정이다. 프로토타입 시연 시에는 이 구조적 로드맵을 구두로 설명하는 것으로 충분하다.
## 3. 기능 명세(더미 스펙 중심)
모든 데이터는 별도의 Backend 서버 없이 `LocalStorage`와 `Mock JSON 파일`에 의존하는 Client-Side 형태로 구현하여 속도와 시연성을 극대화한다.

### 1. 로그인 및 권한 라우팅 (인증 로직 배제)
- **명세:** 아이디/비밀번호 입력란은 존재하지만 아무 값이나 넣어도 무조건 로그인이 승인된다.
- **시연 핵심:** 로그인 화면 하단에 **[학생으로 로그인], [학부모로 로그인], [강사로 로그인] 등 권한별 퀵-버튼(Quick Button)**을 배치한다. 누르는 즉시 해당 역할의 대시보드로 이동하게 하여, 심사위원 앞에서 권한별 화면을 순식간에 스위칭하며 보여준다.

### 2. 학생 / 학부모 뷰 (Data Input & Display)
- **학습 타이머:** `setInterval`을 이용한 단순한 프론트엔드 스톱워치로 뷰를 구성한다. 
- **출결/기록 등록:** 폼(Form)에 입력 후 저장 버튼을 누르면 DB로 가는 것이 아니라, 자바스크립트 배열 상태(State)에 PUSH 되어 바로 아래 목록에 렌더링되도록 처리한다. 
- **차트 표출:** '자기주도지수'나 '주간 학습 시간'은 실제 계산값이 아니라 `[85점, 92점, 78점]`과 같은 화려한 Mock Array 데이터를 기반으로 차트 라이브러리가 즉시 렌더링되게 만든다.

### 3. 교육자 (강사/매니저/컨설턴트) 뷰 (매트릭스 구조 시각화)
- **대시보드 리스트:** 3~5명의 가상 학생 프로필(김철수, 이영희 등)을 하드코딩해 둔다.
- **위험 감지 마크 시연:** 김철수 학생 옆에 빨간색 경고 아이콘(🚨 장기 결석 의심)을 박아두고, 클릭 시 모달창이 뜨며 '상담 코멘트'를 남기는 UI 플로우만 정상 작동하게 만든다. (코멘트를 남기면 State가 업데이트되어 경고창이 사라짐)

### 4. 자동 생성 리포트 기능 (PDF 출력 흉내내기)
- **명세:** 서버 사이드 PDF 생성 로직(Puppeteer 등)을 쓰지 않는다.
- **시연 핵심:** '리포트 인쇄' 버튼을 누르면, 미리 예쁘게 A4 비율로 CSS가 짜여진 `ReportTemplate` 컴포넌트 페이지를 띄우고 브라우저 기본 기능인 `window.print()` 함수를 호출하여 브라우저 인쇄창을 띄워주는 선에서 마무리한다. 심사위원에게는 "클릭 한 번으로 리포트가 출력됩니다" 라고 시연하면 된다.
