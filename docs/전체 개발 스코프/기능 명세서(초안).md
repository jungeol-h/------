# 학습관리 앱 기능 명세서(초안)

## 시스템명
**나르샤** (가칭)
- 부제: 안동시 자기주도 학습 센터 NAVI
- 운영 기관: 안동시 장학회

## 개발 목적
1. 안동 [[학생]]들이 각자의 현재 학습 상태를 정확히 파악하게 하여 자기주도학습을 유도한다.
2. [[관리자]], [[강사]], [[컨설턴트]], [[학습매니저]]가 맞춤형 코칭을 학생에게 지원할 수 있게 한다.
3. 프로토타입 개발을 통해 곧 제출할 사업 지원서에 동작하는 웹 프로토타입 링크를 첨부해 설득력을 높인다.

## 개발 유의사항
현재 개발이 확정된 것이 아니고 상급자에게 사업 추진을 어필하기 위해 웹/앱을 만드는 것이다. 따라서 모든 기능을 완벽하게 구현할 필요가 없다. 실제 개발은 다른 개발팀에서 진행할 수 있기 때문에 최대한 단순하고 빠른 구조로 개발해야 한다.

## 시스템 운영 대상
크게 학습자와 교육자로 나뉜다.

### 학습자
주로 자신의 출결, 학습, 과제, 진로 탐구, 마인드를 기록하고 교육자의 피드백을 확인한다.
1. [[학생]]
2. [[학부모]]

### 교육자
1. [[강사]]
2. [[학습매니저]]
3. [[진로진학 컨설턴트]]
4. [[관리자]]

## 시스템 핵심 기능
1. 출결 기록/관리
2. 학습 기록/관리 (타이머, 과목별 기록, 자기주도지수 산정)
3. 과제 기록/관리 (출제자 표시, 파일 업로드, 미완료 경고)
4. 진로 탐구 기록/관리 (보고서 진행률, 적성 진단, 로드맵)
5. 마인드 기록/관리 (자가 점검 -5~+5점, 3줄 일기)
6. 상담 기록/관리 (코칭 코멘트, 알림 연동)
7. 통계 및 대시보드 (역할별 맞춤 시각화)
8. 운영 리포트 (관리자용 인쇄/PDF)

## 개발 우선순위

### 0단계: 기존 레포 정리 (완료)
기존에는 학습 진단 사이트를 구현했다. 해당 코드를 `src/legacy/`로 이동하고 신규 플랫폼을 `src/platform/`에 구축 완료.

### 1단계: 1차 프로토타입 (완료 — 2026-04-24)
사업 심사위원 시연용. 학생/학습매니저/관리자 3개 역할, Mock 데이터 + LocalStorage, 핵심 시연 시나리오 동작 완료.

### 2단계: 2차 프로토타입 (진행 중)
클라이언트 피드백 반영. 아래 항목들을 추가/수정.

#### 앱 명칭 및 브랜딩 변경
- 타이틀: "나르샤"
- 부제: "안동시 자기주도 학습 센터 NAVI"
- 하단: "안동시 장학회"
- 기존 "안동형 자기주도학습·진로성장 관리 시스템" 텍스트 전체 교체

#### 학생 화면 변경사항
- **홈 탭:** 오늘의 마인드를 날씨 이미지로 표시 / 자기주도 학습지수와 동일 크기로 오늘의 학습(과목+시간 가로형 막대그래프) / 과제 현황 미완료 붉은 경고
- **학습 탭:** 과목 목록 13종으로 확장 (국어, 영어, 수학, 과학, 사회, 도덕, 역사(한국사), 기술가정, 한문, 정보, 교양 독서, 진로 독서, 진로 탐구) / TODO 리스트 메뉴 추가 / 학습 기록을 파이 차트로 통합 표시
- **과제 탭:** 과제 출제자 이름 표시 / 마감 날짜+시간 별도 표시 / 과제 제출하기(파일 업로드) 버튼 추가
- **마인드 탭:** 제목 "마인드 체크"로 변경 / 3항목(-5~+5점) 자가 점검 / "제 얘기 좀 들어주세요." 입력란 / 3줄 일기 탭 분리 (칭찬/반성/다짐)
- **진로 탭:** 제목 "진로 설계"로 변경 / 진로 탐구 보고서 진행률 막대 그래프 (주제선택→완성 8단계)

#### 학습매니저 화면 변경사항
- **홈 탭:** 담당 학생 현황을 "학습 총 시간 / 과제 미수행 / 상담 누적 회차" 3항목으로 변경 / 각 항목 클릭 시 세부 내용 확인 가능

#### 관리자 화면 변경사항
- **홈 탭:** "안동형 자기주도학습 관리시스템" 텍스트 제거 / 인원관리(재적/탈퇴/현인원) 추가 / 탈퇴 버튼 → 탈퇴생 정보 확인 / 상담 현황, 코칭 현황, 컴플레인 메뉴 추가 / 만족도 조사 메뉴 추가 / 학교 현황 섹션 제거
- **통계 탭:** 학습량, 과제 이행도, 마인드 총점 그래프 유지 / 학습 참여도 추이 제거 / 월간 출석률 → "월간 센터 이용시간"으로 변경

### 3단계: 실제 개발
한 달 정도의 개발 시간이 주어진다. 명세서의 모든 기능을 정상 구현하는 것이 목표다.

### 4단계: 고도화
기능 안정성, 기능 고도화, 보안 취약점 보완이 필요하다.
