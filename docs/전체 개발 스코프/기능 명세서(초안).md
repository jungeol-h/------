# 학습관리 앱 기능 명세서(초안)
## 시스템명
안동형 자기주도학습·진로성장 관리 시스템(가칭)
## 개발 목적
1. 안동 [[학생]]들이 각자의 현재 학습 상태를 정확히 파악하게 하여 자기주도학습을 유도한다.
2. [[관리자]], [[강사]], [[컨설턴트]], [[학습매니저]]가 맞춤형 코칭을 학생에게 지원할 수 있게 한다.
3. 프로토타입 개발을 통해 곧 제출할 사업 지원서에 동작하는 웹 프로토타입 링크를 첨부해 설득력을 높인다.
## 개발 유의사항
현재 개발이 확정된 것이 아니고 상급자에게 사업 추진을 어필하기 위해 웹/앱을 만드는 것이다. 따라서 모든 기능을 완벽하게 구현할 필요가 없다. 실제 개발은 다른 개발팀에서 진행할 수 있기 때문에 최대한 단순하고 빠른 구조로 개발해야 한다.
## 시스템 운영 대상
크게 학습자와 교육자로 나뉜다.
### 학습자
주로 자신의 출결, 학습, 과제, 진로 탐구, 마인드를 기록하고 교육자의 피드백을 확인한다.
1. [[학생]]
2. [[학부모]]
### 교육자
1. [[강사]]
2. [[학습매니저]]
3. [[진로진학 컨설턴트]]
4. [[관리자]]
## 시스템 핵심 기능
1. 출결 기록/관리
2. 학습 기록/관리
3. 과제 기록/관리
4. 진로 탐구 기록/관리
5. 마인드 기록/관리
## 개발 우선순위
지금은 1단계: 프로토타입에만 집중해야 한다.

### 0단계: 기존 레포 정리
기존에는 [[CLAUDE(Deprecated)]]를 바탕으로 진단 사이트를 구현했다. 하지만 이 진단 사이트가 이번 기획에 통합된다. 이번 기획은 구조가 확장되는 것이기에 레포 리팩토링이 먼저 이루어져야 하지 않나 싶다. 하지만 빠른 프로토타입(이제 이틀 남음) 관점에서는 핵심 로직만 들어내고 레포를 초기화해 새로 개발하는 것도 방법이겠지만 개발자의 판단에 따라 진행한다.

### 1단계: 프로토타입
다음주까지 더미로 유저스토리 시연이 가능한 정도로 개발이 필요하다. 일단 심사위원이 들어왔을 때 모든 기능이 어떻게 동작하는지 알기만 할 수 있는 정도의 프로토타입이 필요하다.

### 2단계: 실제 개발
한 달 정도의 개발 시간이 주어진다. 명세서의 모든 기능을 정상 구현하는 것이 목표다.
### 3단계: 고도화
기능 안정성, 기능 고도화, 보안 취약점 보완이 필요하다.
